Executive Summary

품질을 개별 테스트가 아닌 운영 가능한 체계로 — 전문 QA 조직의 단일 진실 공급원.

QA Issue Dashboard는 게임 라이브 운영 조직의 QA 팀이 매일 사용하는 통합 품질 관제 시스템이다. 제품 출시 전 검증(Pre-Launch QA), 라이브 서비스 장애 대응(OPS), 마켓 검수(Market Review), 월간 품질 지표 관리, 디바이스 Fleet 운용, 팀 내 협업까지 — QA 운영의 전 단계를 하나의 인터페이스로 포괄한다.

파편화된 JIRA 티켓 · 품질 지표 스프레드시트 · Confluence 장애 리포트 · 디바이스 대여 장부 · 퀘스트 협업 보드를 하나의 대시보드로 묶어, 어제의 품질 이슈 → 오늘의 우선순위 → 내일의 리스크를 한눈에 파악하도록 설계되었다.

품질 측정(검출률 · MTTR · 핫픽스 주기)과 품질 운영(이슈 추적 · 장애 대응 · 팀 협업)을 분리하지 않고 동일한 사실(Fact) 위에서 의사결정이 이루어지는 환경을 제공한다. 이는 단순한 BI 대시보드가 아니라, 품질 경영 실행을 위한 관제탑으로 기능한다.

01
개요 Product Identity & Design Philosophy

Why제품 정체성

QA Issue Dashboard는 게임 라이브 운영 조직의 QA 팀을 위한 통합 관제 시스템이다. 기존에 JIRA · 스프레드시트 · Wiki · 장부 · Slack로 분산되어 있던 품질 운영 데이터를 단일 웹 인터페이스로 집약한다.

운영자가 매일 첫 커피와 함께 여는 대시보드. 매 분기 임원 리뷰에서 열리는 리포트. 매달 서비스 장애 복기에서 펼쳐지는 타임라인. 이 모든 일상이 하나의 제품 안에 담겨 있다.

How운영 데이터 규모

  • 퀘스트(메인) 42건 · 서브 17건
  • 에피소드 8건
  • 길드 6개 · 멤버 52명
  • 디바이스 대여 누적 123건 (진행 중 72)
  • 프론트엔드 TSX/TS 파일 124개
  • 백엔드 Python 모듈 31개

01Editorial

에이전시급 흑백 에디토리얼 톤. Playfair Display + Source Sans 3 · 0px radius. "AI 티 안 나는" 절제된 타이포그래피로 정보 위계를 연출한다.

02Zero-Install

Python stdlib HTTP 서버와 SQLite만으로 구동. 컨테이너 · 메시지큐 · 분산DB 없이 단일 프로세스에 모든 라이브 데이터를 통합한다.

03Real-Time

모든 지표는 원천 시스템에서 직접 수집, 중간 저장은 캐시 계층으로 분리. 어제의 스냅샷이 아닌 지금 이 순간의 현실을 보여준다.

02
핵심 기능 Six Domains, One Interface

대시보드는 6개의 메인 탭으로 구성된다. 각 탭은 독립된 도메인이지만 공통 레이아웃(사이드바 · 필터 · 카드)을 공유해 사용 경험이 일관된다.

01
Tab · JIRA

이슈 통합 + Market Review

4개 게임 프로젝트의 JIRA 이슈를 실시간 프록시하여 서머리 · 추이 · 분포 · 리스트로 시각화. Market Review(PRH) 서브 섹션은 마켓 검수 데이터 + 반려 이슈를 Wiki 캐시와 결합한다.

  • 프로젝트 · 상태 · 담당자 · 기간 필터가 URL 파라미터와 동기화
  • 블로커 이슈 전용 패널 · 우선순위 도넛 · 월간 추이 차트
  • 이슈 리스트는 Virtual Scroll로 수천 건도 즉시 렌더
02
Tab · QA

품질 지표 (PBI)

SharePoint에 저장된 월간 QA 지표 엑셀을 자동 로드해 검출률 · 테스트 볼륨 · 핫픽스 · 커뮤니티 반응을 통합 지표로 표시. 3축 상호 배타 필터 (월 · 프로젝트 · 타겟)로 일관된 드릴다운을 제공한다.

  • 업데이트 이슈 · 핫픽스 · 앱 패치 · 테스트볼륨 · 커뮤니티 차트
  • 월별 트렌드 + 전월 대비 델타 표시
  • Claude 기반 품질 인사이트 서브 섹션 제공
03
Tab · REPORT

QA 리포트 아카이브

Update QA Report와 Monthly QA Report를 GitHub에 HTML 자산으로 보관하고 iframe으로 임베드. 프로젝트별 최신 리포트를 자동 선택하며, 과거 버전도 날짜 인덱스로 탐색 가능하다.

  • 발행 내역 드롭다운 — month/week 메타데이터
  • 임원 리뷰 전용 풀스크린 뷰어
  • 10분 단위 캐시로 빠른 로드
04
Tab · OPS

서비스 안정성 (장애 + 핫픽스)

라이브 서비스 장애와 핫픽스 이력을 Confluence Wiki에서 수집·파싱하여 MTTR · 장애 등급 분포 · 핫픽스 배포 주기 · 이슈 카테고리를 월별 추이로 추적.

  • 장애 상세 패널 → Wiki 원본 직결
  • 사후 복기(Post-Mortem) 워크플로 단축
  • 월간 인시던트 타임라인 + 핫픽스 이벤트 맵
05
Tab · Device

디바이스 Fleet 관리

QA 팀이 운용하는 테스트 디바이스(스마트폰 · 태블릿)의 재고 · 대여 · 반납 · 연장을 SQLite 기반으로 관리. 대여 FSM: 요청 → 승인 → 사용 → 반납 요청 → 승인/거절 → 회수.

  • 2단계 반납 프로세스 (RENT → RETURN_WAIT → Recover)
  • 실물 반납 전 DB 상태 조기 전환 방지
  • 재고 · 대여 추이 · 분포 차트 별도 트렌드 탭
06
Tab · Quest · 3-Depth

협업 워크플로우 + 길드

사내 QA 팀의 업무 협업을 게임화한 퀘스트 기반 작업 관리 시스템. 계층 구조는 3단계:

Episode — 에피소드 (분기 단위 목표)
  └─ Quest — 메인 퀘스트 (2~4주 단위 작업)
       └─ SubQuest — 서브퀘스트 (1~3일 단위 세부 실행)
  • 낙관적 락 + FSM + 히스토리 감사 로그 3중 게이트
  • 퀘스트 보드 · 현황판 · 길드홀 · 에피소드 관리 서브 섹션
  • 에피소드 완료 시 메인 퀘스트별 기여도(%) 어드민 분배
  • 연장 요청/승인 시 차수(#1, #2…) + 사유 한글 detail 자동 기록

외부 의존성 최소화 철학. 단일 Python 프로세스 + SQLite WAL + JSON 캐시로 운영 부담 제거.

외부 연동

JIRA REST Confluence Wiki SharePoint Excel GitHub Pages Claude API

모든 외부 연동은 환경 변수로 토큰 격리. 호스트명/PIN/API 키는 .env (gitignored) 파일로만 주입.

품질 자동화

TS Guard (TS2304) ErrorBoundary Sourcemap Cache-Control FSM 검증 낙관적 락

Vite 플러그인으로 import 누락(TS2304)을 빌드 차단. 프로덕션 런타임 에러는 원본 파일:라인으로 추적.

Design SystemBlack Ink Editorial

Playfair Display italic (heading) + Source Sans 3 (body). 0px radius로 라운드를 배제하고 정보 위계를 날카롭게 유지. 카드·차트 그림자는 5% 미만으로 정적인 인상. 액션 색상은 Green(정상) · Red(중단/실패) · Purple(보류)로 3색 체계.

  • 퀘스트는 on-write 무효화 (실시간)

데이터 파이프라인 Write-Back Flow · State Transition

외부 원천에서 시작된 데이터는 단방향 5단계를 거쳐 UI에 도달하고, 사용자 액션은 같은 경로를 역방향으로 타고 들어가 낙관적 락 · FSM · 히스토리 3중 게이트를 통과해 상태를 변경한다.

상태 변경 예시 · 퀘스트 승인

Sequence · Quest Approve · 5 steps + 4 returns

Gate 1낙관적 락

version 필드로 동시 수정 conflict 감지. 사용자 A · B가 같은 레코드를 수정하면 후착 트랜잭션은 자동 거부된다.

Gate 2FSM 검증

_check_transition이 퀘스트/에피소드별 허용 전이를 검증. 허용되지 않은 상태 이동 시 400 반환, UI는 토스트 경고.

Gate 3히스토리 기록

모든 상태 변경은 _add_history로 한글 detail(예: 마감일: A → B)이 남는다. 감사 추적 및 운영자 리뷰용.
